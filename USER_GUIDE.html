01什么是 Time Machine

Time Machine 是一个静态记忆网站生成器。它读取你的 Logseq Graph 数据，编译成一个可浏览的静态网站，让你以随机探索的方式重新遇见过去的自己。

核心理念
Time Machine 不是博客，而是一个个人时光机。首页只有一个入口——启动时光机，随机穿越到一段记忆。没有时间轴、没有日历、没有主动检索。记忆需要被「解锁」后才能在记忆馆中回顾。

它适合谁？

📝

Logseq 用户

已经在用 Logseq 记录生活和想法，想要一个更沉浸的回顾方式

🎲

喜欢随机性的人

享受被随机推送一段旧记忆的感觉，而非按时间线浏览

🔒

注重隐私的人

纯静态输出，无后端、无数据库、无追踪，部署在哪都行

02三分钟上手

前置要求

  • Node.js 22+（推荐使用 nvm 或 fnm 管理）
  • 一个 Logseq Graph（至少包含 pages/ 目录和一些 .md 文件）

方式一：从零开始（推荐）

如果你还没有项目目录，用 tm init 一键搭建框架：

  1. 初始化项目 
    # 在任意位置创建新项目
tm init my-memories

# 进入项目目录
cd my-memories

    会自动生成 config.json、.gitignore、graph/ 目录结构和一个示例页面。

  2. 放入你的内容

    把 Logseq 的页面（.md 文件）放到 graph/pages/，图片视频放到 graph/assets/。或者直接修改 config.json 的 logseqPath 指向你的 Logseq Graph：

    {
  "logseqPath": "D:/my-logseq-graph",
  "outputPath": "./dist"
}
  3. 构建 + 预览 
    tm s -o   # 自动 build + watch + 开浏览器

03在 Logseq 中写内容

Time Machine 的最小单位不是 Journal（日记），而是 Event（事件）。当前规则：pages/ 目录下每个 .md 文件就是一个 Event。

页面结构

一个标准的 Logseq 页面长这样：

# 页面标题（可选，会被自动识别为文件名）

date:: 2024-03-15
tags:: 旅行, 日本

这是一段自由文本内容。

可以写 [[双链]] 指向其他页面。
可以用 #标签 标记关键词。

![](../assets/photo.jpg)           # Markdown 图片语法
![[video.mp4]]                      # Logseq Embed 视频语法

- 列表项也是支持的
  - 嵌套列表
  - 更多内容

支持的内容类型

类型 语法 说明
页面标题 # 标题 或文件名 文件名即标题，H1 标题如果与文件名相同会被自动跳过
日期 date:: 2024-03-15 页面属性，无则取文件最后修改时间
标签 tags:: 旅行, 日本 或 #旅行 两种写法都支持，会合并去重
双链 [[涩谷]] 指向另一个页面。如果目标页面存在，渲染为可点击链接；不存在则显示为灰色
图片 ![](../assets/photo.jpg) 或 ![[photo.jpg]] 自动提取到图片画廊，正文不再重复显示
视频 ![[video.mp4]] 支持 .mp4 / .mov / .webm 格式
列表 - 列表项 支持嵌套，自动渲染为 HTML 列表
属性 key:: value Logseq 属性语法，date 和 tags 会被解析
最佳实践
  • 给每个页面加上 date:: 属性，这样日期更准确
  • 用 tags:: 或 #标签 标记主题，Time Machine 会自动计算「相关记忆」
  • 多用 [[双链]] 连接相关页面，Time Machine 会自动生成反向链接（Backlinks）
  • 图片放在 assets/ 目录，用 ![[文件名]] 引用

会被自动跳过的页面

以下 Logseq 内部页面不会被解析为 Event：

  • contents.md — Logseq 目录页
  • whiteboards.md — Logseq 白板页
  • journals/ 目录下的所有日记文件

04网站功能

首页

首页只有一个按钮：启动时光机。没有导航栏、没有搜索框、没有文章列表。页面底部显示总记忆数和已解锁数。

点击按钮后，系统会随机选取一段记忆。策略：优先选取尚未解锁的记忆，全部解锁后完全随机。

Event 页面

每段记忆以沉浸式长页面呈现，包含：

📅

日期 + 标题

页面顶部展示，日期高亮显示

📄

正文内容

Markdown 渲染为 HTML，双链解析为可点击链接

🖼️

图片画廊

所有图片自动提取到底部画廊，点击可放大查看

🎬

视频播放器

内嵌视频播放器，支持原生控制

🏷️

标签

所有标签以圆角标签形式展示

🔗

同一天 / 被引用 / 相关记忆

底部展示关联记忆，点击可跳转

关联记忆的三个维度

维度 含义 计算方式
同一天 日期相同的其他 Event 按 date 字段匹配
被引用 (Backlinks) 哪些页面通过 [[双链]] 引用了当前页面 Builder 扫描所有 Event 的 links 字段，建立反向索引
相关记忆 共享标签的其他 Event 按共享标签数量排序，取 Top 5

随机与解锁机制

  • 解锁触发：第一次访问某个 Event 页面时自动解锁（通过 data-event-id 检测）
  • 存储位置：解锁状态保存在浏览器的 localStorage 中，key 为 tm_unlocked
  • 随机策略：优先抽取未解锁的 Event；全部解锁后完全随机
  • 「再次穿越」按钮：Event 页面底部的按钮，点击后随机跳转到另一段记忆
注意
解锁状态存储在浏览器本地。如果换设备、换浏览器或清除浏览数据，解锁记录会丢失。这是设计决策——Time Machine 不依赖任何后端。

记忆馆 (Archive)

记忆馆页面展示所有已解锁的 Event，支持：

  • 搜索：按标题模糊搜索
  • 标签过滤：点击标签 chip 切换过滤
  • 排序：默认按日期倒序
  • 卡片展示：每张卡片显示日期、标题和标签

只有已解锁的记忆才会出现在记忆馆中。未解锁的记忆需要通过「启动时光机」随机遇到后才会出现。

05配置说明

所有配置在项目根目录的 config.json 文件中：

{
  "logseqPath": "./graph",
  "outputPath": "./dist",
  "storage": "local",
  "media": {
    "thumbnailSize": 200,
    "previewSize": 800
  }
}
字段 类型 默认值 说明
logseqPath string "./graph" Logseq Graph 根目录路径，支持相对路径和绝对路径
outputPath string "./dist" 静态网站输出目录
storage string "local" 存储插件：local（本地文件系统）/ r2（Cloudflare R2，占位）/ oss（阿里云 OSS，占位）
media.thumbnailSize number 200 缩略图宽度（像素），用于列表和卡片展示
media.previewSize number 800 预览图宽度（像素），用于点击放大时展示
init 命令
如果你想从头创建一个新的配置文件，可以使用：npx node lib/cli/index.js init --graph ./my-graph --output ./dist

07图片与视频处理

图片处理

Builder 使用 sharp 库自动处理图片，不修改原始文件。每张图片会生成两个版本：

输出文件尺寸格式用途
thumbnail.webp200px 宽WebP列表 / 卡片 / 画廊缩略图
preview.webp800px 宽WebP点击放大时展示的高清图
增量处理
Builder 在清理输出目录时会保留 assets/ 目录。已经生成过 WebP 的图片不会重复处理，大幅加快重复构建速度。如果你修改了原始图片，删除 dist/assets/ 下对应的 WebP 文件即可触发重新处理。

图片引用方式

Time Machine 支持两种图片引用语法：

# 方式一：Markdown 标准语法
![](../assets/photo.jpg)
![描述文字](../assets/photo.jpg)

# 方式二：Logseq Embed 语法
![[photo.jpg]]
![[../assets/photo.jpg]]

两种语法都会被识别，图片会从正文中提取到底部画廊统一展示。

视频处理

视频文件（.mp4 / .mov / .webm）不做转码，直接复制到 dist/assets/videos/ 目录。引用方式：

![[video.mp4]]
![[../assets/clip.mov]]

视频会在页面底部以原生 HTML5 <video> 播放器展示，支持播放控制。

注意
视频文件通常较大，部署时请确保目标平台支持大文件上传。建议对视频进行预压缩（如 HandBrake 压缩为 H.264 + 720p）。

09常见问题

tm g 后显示 Events: 0？
检查 config.json 中的 logseqPath 是否指向正确的 Graph 目录。确认该目录下有 pages/ 文件夹，且其中包含 .md 文件。可以用 tm g -d（dry-run）查看解析详情。
图片显示不出来？
确认图片文件确实存在于 assets/ 目录。Time Machine 会尝试两种路径解析方式：先在 Graph 根目录查找，再在 assets/ 子目录查找。如果图片路径包含 ../ 前缀，会被自动剥离后重新解析。
双链点击后跳转到错误页面？
双链 [[涩谷]] 会尝试匹配 pages/涩谷.md 对应的 Event。如果目标页面不存在，会显示为灰色文字（不可点击）。确保双链文本与页面文件名完全一致（大小写不敏感）。
换了浏览器/设备后解锁记录消失了？
这是预期行为。解锁状态存储在浏览器 localStorage 中，不跨设备同步。Time Machine 的设计理念是无后端、无追踪——解锁记录是个人浏览痕迹，不属于数据本身。
修改了 Logseq 内容后怎么更新网站？
最简单的方式：用 tm s 启动服务器，它默认开启 watch 模式，修改文件后刷新浏览器即可。如果不用 watch，手动 tm g 重新构建即可。Builder 会清理旧的 HTML 文件但保留 assets/ 目录（增量图片处理），所以重复构建很快。
tm s 和 tm g -w 有什么区别？
tm s = build + serve + watch 三合一，启动后既能预览又能自动重建。tm g -w = 只 build + watch，不启动服务器。如果你需要配合自己的 HTTP 服务器（如 nginx），用 tm g -w；如果只是本地预览，直接 tm s。
如何强制重新处理所有图片？
执行 tm clean 删除整个 dist/ 目录（包括缓存的 WebP 图片），然后 tm g 重新构建。这会从头处理所有图片。
能不能把日记 (journals) 也作为 Event？
当前版本不支持。Journal 文件不作为 Event 来源。如果你想展示日记内容，建议在 Logseq 中为每条日记创建一个对应的 Page，或者修改 Parser 源码（src/parsers/logseq/parser.ts）来扩展解析逻辑。
支持 Obsidian 吗？
当前仅支持 Logseq。Parser 是插件化的，未来可以添加 Obsidian Parser。由于 Obsidian 也使用 Markdown + 双链语法，适配工作量不大。
如何自定义网站样式？
网站的 CSS/JS/HTML 模板都在 src/renderers/default/templates.ts 中。你可以直接修改这个文件，然后重新 npm run build && tm g。Renderer 也是插件化的，未来可以开发自定义主题 Renderer。
build 报错 "sharp" 相关错误？
sharp 是原生模块，可能需要根据你的操作系统和 Node.js 版本重新编译。尝试 npm rebuild sharp。如果仍然失败，请检查 Node.js 版本是否 >= 22。
